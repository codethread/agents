# RFCs

Proposal and decision records for ideas that are not yet specs, plans, or tasks.

Use RFCs to explore meaningful uncertainty before a durable domain contract or implementation approach has been chosen. Accepted RFCs should feed long-lived decisions into `devflow/specs/` and build strategy into feature-local `devflow/feat/<feat-name>/` plans and tasks. After implementation, the spec should be sufficient for future work; the RFC remains decision history, not required operational documentation.
